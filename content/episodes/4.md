## Descrizione

Prima puntata registrata in uno studio vero con regista e tutto, e ovviamente l'abbiamo dedicata al CSS. Abbiamo messo sul banco degli imputati Bootstrap con i suoi componenti preconfezionati che rendono tutti i siti uguali, e abbiamo celebrato l'arrivo di Tailwind CSS con il suo approccio utility-first. La metafora del castello? Vanilla CSS è la sabbia nel secchiello (faticoso), Bootstrap è il castello Mattel già pronto (comodo ma tutti uguali), Tailwind sono i Lego (libertà totale di costruzione).

## Takeaway

- L'approccio utility-first di Tailwind si basa sul principio di singola responsabilità: ogni classe CSS fa una cosa sola (bordo arrotondato, ombra, colore testo)
- Bootstrap porta il concetto di componente nel CSS, ma concettualmente il CSS dovrebbe solo descrivere l'estetica, non la funzionalità (quella è compito dei web components)
- Con PurgeCSS si può ridurre Tailwind da 70KB a dimensioni ridicole eliminando tutte le utility non utilizzate
- La direttiva @apply permette di creare componenti CSS custom combinando utility, utile per ridurre la verbosità dell'HTML
- Tailwind fornisce palette di colori coerenti out-of-the-box, risolvendo il problema del full-stack che sceglie i colori con la pipetta abbassando la luminosità a caso

## Bold Opinion

- jQuery come dipendenza obbligatoria di Bootstrap è inaccettabile: vogliamo scegliere liberamente il framework JavaScript senza accoppiamenti prestabiliti
- Il fatto che Bootstrap ti costringa a scrivere vanilla CSS per fare override quando esci dai canoni prestabiliti è la dimostrazione che il modello a componenti CSS non scala
- Le dimensioni contano, ma Tailwind vale ogni KB in più perché la libertà creativa non ha prezzo (e poi con PurgeCSS il problema si risolve)
