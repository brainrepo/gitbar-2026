## Descrizione

Ci siamo seduti con Mattia Tommasone per parlare di Kotlin, il linguaggio che vorremmo Java fosse nel 2021. Da un team Kotlin shop al billion dollar mistake risolto con variabili non-nullable per default, dalle coroutines che battono i thread Java alle data class che fanno piangere Lombok, esploriamo perché JetBrains ha creato Kotlin (Scala compila troppo lento), come Google ha fatto Kotlin first-class su Android, e perché compilare in 2 secondi invece che 20 ti salva da 3 ore di tunnel Wikipedia. Tra type system statico che ti controlla mentre scrivi e IntelliJ che converte Java in Kotlin con un click, scopriamo che la JVM non è più quella vorace di RAM che ricordavamo.

## Takeaway

- **Kotlin è Java come vorrei fosse nel 2021**: ergonomia fantastica, puoi scrivere Java in Kotlin se vuoi. IDE converte automaticamente, zero curva di apprendimento
- **Billion dollar mistake risolto**: variabili non-nullable per default. Se vuoi nullable, scrivi Type? esplicitamente. No più null pointer exception ovunque
- **Coroutines sono lightweight threads**: programmazione asincrona moderna. Flow e Channel come Go. Backend salva castagne dal fuoco, eredità da Android
- **Data class native**: getter, setter, equals, toString gratis. Lombok piange. Google lo fa nativamente, JetBrains sa fare IDE
- **Type system statico ti salva**: errori scoperti a compile time costano infinitamente meno. Se compili, è quasi corretto
- **Compilazione veloce come Java**: 2 secondi, non 20. Sotto la soglia di distrazione. Scala troppo lento, Kotlin ha tempi paragonabili a Java

## Bold Opinion

- **JVM non è più voracissima**: sì, serve più RAM di Node o Go compilato. Ma oggi Heroku slider risolve, non apri server a mano con cacciavite
- **Fun vs Interface non è inconsistenza**: Int è un tipo, non puoi usarlo per interfacce. Critiche superficiali da chi non conosce il linguaggio
- **1.3 to 1.4 breaking changes fanno male**: linguaggio giovane, alcune feature core instabili. Doloroso per librerie con adoption grossa
- **Kotlin è primo su Android, non solo back-end**: Google dice Kotlin first. Documentazione, talk, tutto in Kotlin. Java è il passato
